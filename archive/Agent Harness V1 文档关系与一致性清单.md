## 权威术语基线

1. `public:workflow` 统一表示可复用的 `workflow_definition` / template，不表示运行态 `workflow_instance`。
2. `workflow_instance`、`workflow_stage`、`checkpoint`、`execution_session` Day 1 默认都是私有运行态对象。
3. `policy_snapshot` 的权威物理关联是 `workflow_instance.policy_snapshot_id`，不再通过反向硬 FK 形成循环依赖。
4. 渠道原始附件先进入 `artifact_object(raw_attachment)`，只有在需要长期检索或抽取时才提升为 `document`。
5. 未绑定身份只能进入接入层绑定前流程，不能直接进入 Workflow `waiting_user`。
6. 所有外部能力通过 Provider Adapter 接入，支持供应商切换与降级。
7. 错误分为 Transient（可重试）、Permanent（不可重试）、System（系统级）三类。
8. API 采用 URL 路径版本策略，版本生命周期为 Current → Deprecated → Sunset → Retired。

## Agent 自动校验条目（必须全部通过）

1. 术语一致性
   - `public:workflow` 在所有文档均表示 `workflow_definition/template`。
   - 运行态对象不被描述为公共区资源。
   - Provider 相关术语统一使用 Adapter 模式。
   - 错误分类术语统一使用 Transient/Permanent/System。

2. 权限一致性
   - `policy_snapshot` 关联路径一致（以 `workflow_instance.policy_snapshot_id` 为权威）。
   - 未绑定身份不进入 Workflow `waiting_user`。
   - 所有外部能力访问都经过 policy 校验。

3. 数据模型一致性
   - `workflow_stage` / `checkpoint` / `execution_session` 字段与契约一致。
   - `artifact_object` 与 `document` 边界一致。
   - Provider 配置结构与 Adapter 接口一致。

4. 状态与恢复一致性
   - `waiting_user/blocked/paused/failed` 定义一致。
   - replay 边界一致且默认只读。
   - 错误重试与降级规则与状态机一致。

5. 计划一致性
   - 里程碑前置约束明确：M0 通过后才允许开发。
   - 文档依赖图与阅读顺序可执行。
   - API 版本生命周期与废弃策略一致。
