# 全面评估与执行报告

## 1. 技术可行性评估

| 优化项 | 技术难度 | 依赖 | 可行性 |
|--------|---------|------|--------|
| TTLMap | 低 (标准 TS 泛型类) | 无 | 100% |
| LlmClient | 低 (封装 fetch) | 无 | 100% |
| PoolManager | 低 (延迟初始化单例) | pg 原生 Pool | 100% |
| ChannelAdapter | 中 (需完整实现飞书/企业微信 API) | 现有 env 变量 | 100% |
| IntentClassifier | 中 (两层分类逻辑) | LlmClient | 100% |
| createMicroService | 中 (需兼容各服务路由) | http 原生 | 100% |

## 2. 风险评估

| 风险 | 级别 | 缓解措施 |
|------|------|----------|
| API 签名不兼容 | 低 | 保留 verifyFeishuSignature/verifyWecomSignature 作为兼容层 |
| DB 连接池配置差异 | 低 | PoolManager 透传所有 pg.Pool options |
| LLM prompt 行为变化 | 低 | IntentClassifier 使用完全相同的 prompt 内容 |
| 回滚困难 | 低 | 所有修改在 fork 分支, master 不受影响 |
