# 最终重构计划 (Fork版本)

> **版本**: v2.1-final | **日期**: 2026-05-10
> **原则**: 斟酌损益，只做收益明确、风险可控的优化

---

## 一、决策原则

基于权衡评估，采用以下决策原则：

| 原则 | 说明 |
|------|------|
| **收益>损失** | 只做净收益明确的优化 |
| **风险可控** | 不做高风险或需大量补充的优化 |
| **渐进式** | 分阶段实施，每阶段可独立验证 |
| **保留原版能力** | 不损失核心功能 |

---

## 六、重构优先级排序

```
优先级 = 收益 / (风险 × 工期)

Phase 1: 基础设施 → 优先级最高（收益高，风险低）
Phase 2: 渐进替换 → 优先级高（收益中，风险低）
Phase 3: IntentClassifier → 优先级中（收益中，风险中）
Phase 4: 模块化 → 优先级低（收益高，风险高）
```

---

## 七、风险控制措施

| 风险 | 控制措施 |
|------|----------|
| 功能回归 | 每阶段运行原版集成测试 |
| 性能下降 | 每阶段做性能基准测试 |
| 回滚困难 | 每阶段独立提交，可单独回滚 |
| 文档不一致 | 每阶段更新 DEV-08 图谱 |
