# 文档 28：配置管理设计 v1.0

## 28.1 文档目的

本文件定义 Agent Harness V1 的配置管理体系，覆盖：

- 配置层级与优先级
- 环境变量规范
- 配置文件结构
- 敏感配置管理
- 配置热更新机制
- 配置验证与默认值

## 28.2 配置层级

### 28.2.1 优先级规则

| 层级 | 来源 | 优先级 | 示例 |
|------|------|--------|------|
| 1 | 代码默认值 | 最低 | `DEFAULT_PORT = 3000` |
| 2 | 配置文件 | 低 | `config/default.yaml` |
| 3 | 环境特定配置文件 | 中 | `config/production.yaml` |
| 4 | 环境变量 | 高 | `SERVER_PORT=8080` |
| 5 | 命令行参数 | 最高 | `--port=8080` |

### 28.2.2 配置合并规则

```
最终配置 = 代码默认值 
         ⊕ 配置文件默认值 
         ⊕ 环境特定配置 
         ⊕ 环境变量 
         ⊕ 命令行参数
```

## 28.3 环境变量规范

### 28.3.1 命名规范

| 规则 | 示例 | 说明 |
|------|------|------|
| 全大写 | `DATABASE_URL` | 标准约定 |

```typescript
const REQUIRED_CONFIGS = [
  'DATABASE_URL',
  'REDIS_URL',
  'OPENAI_API_KEY',
];

function checkRequiredConfigs(): void {
  const missing = REQUIRED_CONFIGS.filter(key => !process.env[key]);
  if (missing.length > 0) {
    throw new Error(`Missing required configs: ${missing.join(', ')}`);
  }
}
```

## 28.7 配置热更新

### 28.7.1 支持热更新的配置

| 配置 | 热更新 | 说明 |
|------|--------|------|
| 日志级别 | ✅ | 动态调整日志级别 |
| 限流阈值 | ✅ | 动态调整限流 |
| 功能开关 | ✅ | 功能开关切换 |
| 数据库连接 | ❌ | 需要重启 |
| Redis 连接 | ❌ | 需要重启 |

### 28.7.2 热更新机制

```typescript
class ConfigManager {
  private config: Config;
  private watchers: ConfigWatcher[] = [];

  async reload(): Promise<void> {
    const newConfig = await this.loadConfig();
    const diff = this.diffConfig(this.config, newConfig);
    
    if (diff.requiresRestart) {
      this.logger.warn('Config change requires restart', {
        changedKeys: diff.changed,
        reason: diff.restartReason
      });
      this.emitConfigChangeEvent(newConfig, diff);
      return;
    }

    this.config = newConfig;
    this.notifyWatchers(diff);
  }

  watch(keys: string[], callback: ConfigCallback): void {
    this.watchers.push({ keys, callback });
  }

  private notifyWatchers(diff: ConfigDiff): void {
    this.watchers.forEach(w => {
      if (w.keys.some(k => diff.changed.includes(k))) {
        w.callback(this.config);
      }
    });
  }
}
```

## 28.9 配置文档化

### 28.9.1 配置说明模板

```markdown
## DATABASE_URL

**类型**: String (URI)  
**必填**: 是  
**默认值**: 无  
**环境变量**: `DATABASE_URL`

PostgreSQL 数据库连接字符串。

格式: `postgresql://<user>:<password>@<host>:<port>/<database>`

示例:
```
DATABASE_URL=postgresql://agent_harness:password@localhost:5432/agent_harness
```

注意事项:
- 生产环境必须启用 SSL
- 密码不要包含特殊字符或需要 URL 编码
```

## 28.10 Day 1 验证用例

1. 所有必填配置缺失时服务启动失败。
2. 配置验证失败时返回明确错误信息。
3. 环境变量正确覆盖配置文件值。
4. 敏感配置不从日志中泄露。
5. 日志级别热更新立即生效。
6. 功能开关切换正确生效。
