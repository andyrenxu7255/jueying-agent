# 文档 31：错误处理与降级策略 v1.0

## 31.1 文档目的

本文件定义 Agent Harness V1 的错误处理规范与降级策略，覆盖：

- 错误分类与编码
- 错误传播与处理
- 重试策略
- 降级规则
- 熔断与限流
- 错误恢复与告警

## 31.2 错误分类体系

### 31.2.1 错误层级

```
Error
├── TransientError (可重试)
│   ├── ProviderTimeoutError
│   ├── RateLimitError
│   ├── ConnectionError
│   └── ServiceUnavailableError
│
├── PermanentError (不可重试)
│   ├── ValidationError
│   ├── AuthenticationError
│   ├── AuthorizationError
│   ├── NotFoundError
│   └── ConflictError
│
└── SystemError (系统级)
    ├── ConfigurationError
    ├── DatabaseError
    └── InternalError
```

### 31.2.2 错误码规范

| 前缀 | 类别 | 示例 |
|------|------|------|
| `CHANNEL_*` | 渠道接入 | `CHANNEL_PAYLOAD_INVALID`、`CHANNEL_DELIVERY_FAILED` |
| `IDENTITY_*` | 身份绑定 | `IDENTITY_NOT_BOUND` |
| `POLICY_*` | 权限策略 | `POLICY_DENIED`、`POLICY_SNAPSHOT_MISSING`、`POLICY_HASH_MISMATCH`、`POLICY_SCOPE_INVALID` |
| `WORKFLOW_*` | Workflow 治理 | `WORKFLOW_PLANNING_FAILED`、`WORKFLOW_NOT_FOUND` |
| `STAGE_*` | 阶段执行 | `STAGE_TIMEOUT` |
| `EXECUTOR_*` | 执行器 | `EXECUTOR_UNAVAILABLE`、`EXECUTOR_TOOL_CHAIN_FAILED`、`EXECUTOR_TEST_FAILED`、`EXECUTOR_TIMEOUT` |
| `RETRIEVAL_*` | 检索 | `RETRIEVAL_DEGRADED`、`RETRIEVAL_TIMEOUT` |
| `FACT_*` | 事实写入 | `FACT_CONFLICT` |
| `ARTIFACT_*` | 对象存储 | `ARTIFACT_NOT_FOUND` |
| `INTEGRATION_*` | 外部集成 | `INTEGRATION_LLM_UNAVAILABLE`（LLM 完全不可用）、`INTEGRATION_LLM_ERROR`（LLM 通用错误，含部分降级）、`INTEGRATION_ALL_PROVIDERS_DOWN`、`INTEGRATION_HERMES_TIMEOUT`、`INTEGRATION_HERMES_CONNECTION_FAILED` |
| `SYSTEM_*` | 系统级 | `SYSTEM_CONFIGURATION_ERROR`、`SYSTEM_DATABASE_ERROR`、`SYSTEM_RATE_LIMIT_EXCEEDED`、`SYSTEM_API_VERSION_RETIRED`、`SYSTEM_API_VERSION_NOT_FOUND` |

> **说明**：`INTEGRATION_LLM_UNAVAILABLE` 表示 LLM 服务完全不可用（所有供应商均失败），`INTEGRATION_LLM_ERROR` 表示 LLM 调用出现可降级错误（如主供应商超时但备选可用）。所有错误码须遵循文档 15 §15.3 定义的前缀规范。

## 31.3 错误处理规则

### 31.3.1 错误传播规则

| 层级 | 处理方式 | 说明 |
|------|----------|------|
| Adapter 层 | 转换为标准错误 | 统一错误格式 |
| Service 层 | 包装并添加上下文 | 保留原始错误链 |
| API 层 | 返回 HTTP 响应 | 映射到 HTTP 状态码 |
| Workflow 层 | 状态迁移 | 进入 `failed` 或 `blocked` |

### 31.3.2 HTTP 状态码映射

| 错误类别 | HTTP 状态码 | 说明 |
|----------|-------------|------|
| Validation | 400 Bad Request | 请求参数错误 |
| Authentication | 401 Unauthorized | 未认证 |
| Authorization | 403 Forbidden | 无权限 |
| NotFound | 404 Not Found | 资源不存在 |
| Conflict | 409 Conflict | 资源冲突 |
| RateLimit | 429 Too Many Requests | 限流 |
| Provider | 502 Bad Gateway | 上游服务错误 |
| Timeout | 504 Gateway Timeout | 超时 |
| Internal | 500 Internal Server Error | 内部错误 |

### 31.3.3 错误处理代码示例

```typescript
class ErrorHandler {
  handle(error: Error, context: ErrorContext): ErrorResult {
    const classified = this.classify(error);

    this.logger.error('Error occurred', {
      code: classified.code,
      message: error.message,
      trace_id: context.trace_id,
      retryable: classified.retryable,
      stack: error.stack
    });

    if (classified.auditable) {
      this.auditWriter.write({
        action: 'error.occurred',
        detail: {
          code: classified.code,
          message: error.message
        }
      });
    }

    return {
      code: classified.code,
      message: this.sanitizeMessage(error),
      retryable: classified.retryable,
      detail: this.extractDetail(error)
    };
  }

  private sanitizeMessage(error: Error): string {
    return error.message
      .replace(/password=\S+/gi, 'password=***')
      .replace(/api_key=\S+/gi, 'api_key=***');
  }
}
```

## 31.4 重试策略

### 31.4.1 重试规则

| 错误类型 | 是否重试 | 说明 |
|----------|----------|------|
| `RateLimitError` | ✅ | 等待后重试 |
| `TimeoutError` | ✅ | 短暂等待后重试 |
| `ConnectionError` | ✅ | 指数退避重试 |
| `ValidationError` | ❌ | 请求本身有问题 |
| `AuthenticationError` | ❌ | 凭证问题 |
| `AuthorizationError` | ❌ | 权限问题 |
| `NotFoundError` | ❌ | 资源不存在 |

### 31.4.2 重试配置

```typescript
interface RetryPolicy {
  max_retries: number;
  initial_delay_ms: number;
  max_delay_ms: number;
  backoff_multiplier: number;
  jitter: boolean;
  retryable_errors: string[];
}

const DEFAULT_RETRY_POLICY: RetryPolicy = {
  max_retries: 3,
  initial_delay_ms: 100,
  max_delay_ms: 30000,
  backoff_multiplier: 2,
  jitter: true,
  retryable_errors: [
    'RateLimitError',
    'TimeoutError',
    'ConnectionError',
    'ServiceUnavailableError'
  ]
};
```

### 31.4.3 重试实现

```typescript
async function withRetry<T>(
  fn: () => Promise<T>,
  policy: RetryPolicy
): Promise<T> {
  let lastError: Error;
  let delay = policy.initial_delay_ms;

  for (let attempt = 0; attempt <= policy.max_retries; attempt++) {
    try {
      return await fn();
    } catch (error) {
      lastError = error;

      if (!policy.retryable_errors.includes(error.constructor.name)) {
        throw error;
      }

      if (attempt === policy.max_retries) {
        break;
      }

      const jitter = policy.jitter ? Math.random() * 0.1 * delay : 0;
      await sleep(delay + jitter);
      delay = Math.min(delay * policy.backoff_multiplier, policy.max_delay_ms);
    }
  }

  throw lastError;
}
```

## 31.5 降级策略

### 31.5.1 降级触发条件

| 条件 | 触发降级 | 说明 |
|------|----------|------|
| 服务连续失败 N 次 | 是 | 熔断器打开 |
| 响应时间超过阈值 | 是 | 性能降级 |
| 资源使用超限 | 是 | 容量降级 |
| 上游服务不可用 | 是 | 依赖降级 |

### 31.5.2 检索降级矩阵

| 失败点 | 降级策略 | 影响 |
|--------|----------|------|
| 结构化查询失败 | 尝试全文检索 | 可信度下降 |
| 全文检索失败 | 保留结构化+向量 | 覆盖减少 |
| 向量检索失败 | 保留结构化+全文 | 语义召回缺失 |
| AGE 图增强失败 | 跳过图扩展 | 关联信息缺失 |
| Rerank 失败 | 使用规则排序 | 排序质量下降 |
| Memory 检索失败 | 跳过 Memory | 经验提示缺失 |

### 31.5.3 LLM 降级矩阵

| 场景 | 降级策略 |
|------|----------|
| 主模型超时 | 切换到备选模型 |
| 主模型限流 | 切换到备选供应商 |
| 所有模型不可用 | Workflow 进入 `blocked` |
| Embedding 失败 | 跳过向量召回 |
| Rerank 失败 | 使用 BM25 排序 |

## 31.6 熔断器

### 31.6.1 熔断器状态

```
         失败率 > 阈值
    ┌─────────────────────┐
    │                     │
    ▼                     │
┌────────┐  超时    ┌────────┐
│ CLOSED │ ───────▶ │  OPEN  │
└────────┘          └────────┘
    ▲                     │
    │                     │ 半开探测
    │    探测成功         │
    └─────────────────────▼
                    ┌──────────┐
                    │ HALF_OPEN│
                    └──────────┘
```

### 31.6.2 熔断器配置

```typescript
interface CircuitBreakerConfig {
  failure_threshold: number;
  failure_rate_threshold: number;
  min_requests: number;
  open_duration_ms: number;
  half_open_max_calls: number;
}

const DEFAULT_CIRCUIT_BREAKER: CircuitBreakerConfig = {
  failure_threshold: 5,
  failure_rate_threshold: 0.5,
  min_requests: 10,
  open_duration_ms: 30000,
  half_open_max_calls: 3
};
```

### 31.6.3 熔断器实现

```typescript
class CircuitBreaker {
  private state: 'CLOSED' | 'OPEN' | 'HALF_OPEN' = 'CLOSED';
  private failures = 0;
  private successes = 0;
  private lastFailureTime: number;

  async execute<T>(fn: () => Promise<T>): Promise<T> {
    if (this.state === 'OPEN') {
      if (Date.now() - this.lastFailureTime > this.config.open_duration_ms) {
        this.state = 'HALF_OPEN';
        this.successes = 0;
      } else {
        throw new CircuitOpenError('Circuit breaker is open');
      }
    }

    try {
      const result = await fn();
      this.onSuccess();
      return result;
    } catch (error) {
      this.onFailure();
      throw error;
    }
  }

  private onSuccess(): void {
    this.failures = 0;
    if (this.state === 'HALF_OPEN') {
      this.successes++;
      if (this.successes >= this.config.half_open_max_calls) {
        this.state = 'CLOSED';
      }
    }
  }

  private onFailure(): void {
    this.failures++;
    this.lastFailureTime = Date.now();
    if (this.failures >= this.config.failure_threshold) {
      this.state = 'OPEN';
    }
  }
}
```

## 31.8 错误恢复流程

### 31.8.1 Workflow 错误恢复

```
错误发生
    │
    ▼
┌─────────────┐
│ 分类错误    │
└─────────────┘
    │
    ├── 可重试错误 ──▶ 重试 ──▶ 成功 ──▶ 继续
    │                      │
    │                      ▼
    │                    失败 ──▶ 检查重试次数
    │                                    │
    │                    ┌───────────────┘
    │                    │
    │                    ▼
    │              未达上限 ──▶ 继续重试
    │              已达上限 ──▶ 进入修复/失败
    │
    ├── 可修复错误 ──▶ 进入 Repair 阶段
    │
    ├── 需要用户 ──▶ 进入 waiting_user
    │
    ├── 外部依赖 ──▶ 进入 blocked
    │
    └── 不可恢复 ──▶ 进入 failed
```

## 31.9 告警规则

### 31.9.1 错误告警

| 告警 | 条件 | 严重性 |
|------|------|--------|
| 错误率异常 | 错误率 > 5% 持续 2 分钟 | warning |
| 错误率严重 | 错误率 > 20% 持续 1 分钟 | critical |
| 特定错误激增 | 特定错误码 5 分钟内 > 100 次 | warning |
| 熔断器打开 | 任意熔断器打开 | warning |
| 全部熔断 | 关键服务全部熔断 | critical |

### 31.9.2 降级告警

| 告警 | 条件 | 严重性 |
|------|------|--------|
| 检索降级 | 降级次数 5 分钟内 > 10 次 | warning |
| LLM 降级 | 主供应商切换 | warning |
| 全部降级 | 所有增强能力不可用 | critical |

## 31.10 Day 1 验证用例

1. 可重试错误正确重试并最终成功或失败。
2. 不可重试错误立即返回，不重试。
3. 熔断器在连续失败后正确打开。
4. 熔断器半开状态正确探测恢复。
5. 限流超限返回正确的 429 响应及 `SYSTEM_RATE_LIMIT_EXCEEDED` 错误码。
6. 检索降级正确记录并返回结果。
7. 错误信息不包含敏感数据。
8. 所有错误码符合文档 15 §15.3 定义的前缀规范。
